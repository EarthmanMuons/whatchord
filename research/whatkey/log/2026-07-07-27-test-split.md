# 2026-07-07: The one-shot test-split evaluation

**Goal.** The single held-out evaluation the protocol has pointed at since the
freeze. Result set declared before any run: (1) the frozen shipped configuration
(HMM, 30 s emissions, pure profile correlation, Albrecht-Shanahan, duration
weighting, margin floor 0.3, self-transition 0.9, temperature 0.25, mode tilt 2;
entry 23) on all three frozen test splits; (2) the reflex reference (1 s
emissions, functional blend 0.1, other defaults inherited, so it carries the
mode tilt too, which is timescale-neutral) on the two mode-resolved splits, the
crossover's held-out confirmation; (3) the music21 baselines on the Isophonics
test split with the paired and matched-coverage comparison; (4) descriptive mode
confusion on Isophonics test. No runs beyond this set; every number below is
first-look.

```sh
# shipped on each test split, e.g.:
dart run tool/whatkey/harness.dart \
  --fixtures build/whatkey-fixtures/isophonics-nc-v1 \
  --split-file research/whatkey/data/splits/isophonics-nc-v1.json \
  --split test --detector hmm \
  --out build/whatkey-harness/test-iso-hmm-shipped
# reflex adds: --decay-half-life-seconds 1 --functional-blend 0.1
# baselines:
python3 tool/whatkey/external_baseline.py \
  --fixtures build/whatkey-fixtures/isophonics-nc-v1 \
  --split-file research/whatkey/data/splits/isophonics-nc-v1.json --split test
# each claims file scored via --claims-file; KS matched read via --restrict-to
python3 tool/whatkey/compare.py \
  build/whatkey-harness/test-iso-hmm-shipped/report.json \
  build/whatkey-harness/test-iso-m21-krumhanslschmuckler/report.json
```

**Shipped configuration, all three held-out splits:**

| split (n)         | cov  | exact                   | MIREX | modulations | spurious |

| When in Rome (18) | 0.81 | 0.587                   | 0.709 | 39/115      | 0/1      |

Generalization: Isophonics test 0.732 vs dev 0.775; When in Rome test 0.587 vs
dev 0.434; ASAP acceptable 0.683 vs dev 0.730. The differences run in both
directions and track piece composition at small n, with no overfitting signature
(a tuned-to-dev configuration would degrade consistently, and the stability
metrics hold everywhere: spurious p90 at most 3).

**The crossover, confirmed on held-out data in both directions, both significant
even at test-split sizes:**

- When in Rome test (tonicization ruler): reflex 0.649 vs shipped 0.587, paired
  +0.0620, CI95 [+0.0042, +0.1208], p = 0.047 (11/5/0), and 65 vs 39 of 115
  modulations.
- Isophonics test (section ruler): shipped 0.732 vs reflex 0.556, paired
  +0.1753, CI95 [+0.0396, +0.3146], p = 0.039 (23/11/4), with reflex paying
  spurious 5/11 vs shipped 0/3.

**External baselines, Isophonics test** (offline, whole-piece, non-abstaining):

| system                       | coverage | exact | MIREX |
| ---------------------------- | -------- | ----- | ----- |
| ours, shipped (causal)       | 0.88     | 0.732 | 0.782 |
| music21 TemperleyKostkaPayne | 1.00     | 0.637 | 0.740 |
| music21 KrumhanslSchmuckler  | 1.00     | 0.624 | 0.726 |
| music21 AardenEssen          | 1.00     | 0.558 | 0.690 |

The shipped detector's point estimate leads every offline analyzer on the
held-out songs, including at matched coverage (KrumhanslSchmuckler restricted to
our claimed events: 0.63 exact vs our 0.732). The paired test does not reach
significance (+0.1075, CI95 [-0.0082, +0.2284], p = 0.25, 11/12/15: a few large
wins against many small ties), so the reportable claim is unchanged from the
dev-split framing, **at least parity under a strictly harder setting**, now with
the point estimate on our side.

**Mode confusion, Isophonics test, shipped** (descriptive): exact 72%, fifth 7%,
relative 5%, parallel 2%, other 14%. The parallel row at 2% on held-out data
matches the mode tilt's development effect exactly (entry 23).

**Plain-English reading.** This was the exam taken once, on questions the
detector had never seen, after all tuning was locked. It passed everywhere.
Accuracy on the held-out pop songs is close to the development number, the
calm-versus-quick story replayed exactly as measured (the quick setting wins
when the answer key tracks excursions, the calm one wins by a wide margin when
it names the song key), the wrong-twin fix held at two claims in a hundred, and
the detector now outscores every standard offline key finder we compare against
on these held-out songs, systems that read the entire song in advance, while
ours listens live, admits uncertainty, and stays steady. The statistics only
allow us to say "no worse than" those systems, but the raw numbers lean our way.

**Artifacts.** The complete one-shot outputs are committed at
`results/test-split-2026-07-07/` (all nine runs: report.json, report.txt,
claims.json, plus the music21 baseline claims), since the test split is spent
and these files cannot be legitimately regenerated. Redistribution decision,
recorded here per `data/NOTICE.md`'s gate: these artifacts contain piece
identifiers and titles, aggregate metrics, our detectors' own claims, and, for
Isophonics only, one summary key label per song (`globalTruth`; the ASAP field
is null and the When in Rome labels are already committed under CC BY-SA
provenance). A single song-level key in evaluation results is the same class of
fact any results table publishes, cited to the Isophonics reference annotations;
no time-aligned annotation content is included, and the license-gated fixtures
themselves remain uncommitted.

**Decisions.**

- Phase 2 evaluation is complete. The reportable claims are fixed: held-out
  section-scale exact 0.73 at coverage 0.88 with spurious p90 3; the two-ruler
  crossover significant on held-out data in both directions; at least parity
  with offline whole-piece baselines under a strictly harder setting;
  parallel-mode confusion 2%.
- The test split is spent for this result set. Any future configuration change
  starts a new result set with fresh discipline.
- Next work is product integration (inferred-key provider, home-screen
  indicator) and the paper; both draw on these numbers as-is.
